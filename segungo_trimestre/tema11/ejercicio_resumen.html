Resumen de XML Schema

La validación XSD sirve para definir las reglas que debe cumplir un archivo XML.

¿Qué es XML Schema?

XML Schema, también llamado XSD, es un lenguaje que permite definir la estructura y las reglas de un documento XML. Sirve para indicar qué elementos y atributos puede tener, en qué orden van y qué tipo de datos usan.

Características principales

  • Es un archivo XML.
  • Usa sintaxis XML, a diferencia de DTD.
  • Su extensión es .xsd.
  • Permite validar documentos XML.

Qué permite definir un XSD

  • Los elementos y atributos que pueden aparecer.
  • El número y orden de los elementos.
  • Los tipos de datos.
  • Valores obligatorios, opcionales o por defecto.

Tipos de elementos

Elementos simples: contienen solo texto o un dato simple, como un nombre o un número.

Elementos complejos: contienen otros elementos o atributos.

Tipos de datos

En XSD se pueden usar tipos ya definidos, por ejemplo:

  • xs:string
  • xs:integer
  • xs:decimal
  • xs:boolean
  • xs:date

Restricciones

XSD también permite poner restricciones a los datos. Por ejemplo, limitar un número entre 1 y 10, fijar una longitud exacta o permitir solo ciertos valores.

  • minInclusive y maxInclusive
  • length, minLength, maxLength
  • enumeration
  • pattern

Atributos

Los atributos se definen con <xs:attribute> y pueden ser obligatorios, opcionales o tener valores fijos.

Asociación entre XML y XSD

Para validar un XML con su esquema, se enlaza así:

xsi:noNamespaceSchemaLocation="archivo.xsd"

Documentación del XSD

Un esquema XSD puede incluir documentación con xs:annotation, xs:documentation y xs:appinfo.

Resumen final

XSD es importante porque ayuda a comprobar que un XML esté bien construido y cumpla unas reglas. Gracias a esto, la información se puede compartir entre programas de forma más segura y ordenada.
